Resumen del Proyecto: Sistema de Gestión de Trámites de Gestión Empresarial

Este documento describe el contexto de negocio, los usuarios y el flujo del sistema.
Para reglas técnicas ver: agent.md
Para conexiones entre archivos ver: docs/DEPENDENCIES.md

🎯 Objetivo del Proyecto
Crear un sistema profesional de gestión de solicitudes para el departamento de Gestión Empresarial, que apoya a los contadores en diversas diligencias legales y administrativas ante entidades como la DIAN, Cámara de Comercio, etc.

📌 Origen
El proyecto nació como una plantilla de tickets para un departamento de IT ("IT Command"). Se adaptó completamente para el departamento de Gestión Empresarial, cambiando la terminología, los campos del formulario y los tipos de trámites. La transformación ya fue completada.

👥 Usuarios del Sistema

1. Colaboradores (empleados/solicitantes):
   - Acceden al portal (ruta `/portal/:area`)
   - Envían solicitudes de trámites seleccionando su área y tipo
   - Pueden ver el historial de sus propias solicitudes
   - Ven el estado del personal y los sistemas del área seleccionada

2. Gestoras y Administradores:
   - Acceden al dashboard administrativo (ruta `/dashboard/:area` y `/database/:area`)
   - Ingresan a través de la ruta protegida (`/login`) con sistema local encriptado
   - Reciben notificaciones en tiempo real cuando llega una nueva solicitud
   - Gestionan los tickets: asignan responsable, cambian estado y prioridad
   - Publican su estado personal (disponible, en reunión, ocupada)

🔄 Flujo Principal
Colaborador abre portal → Selecciona área y trámite → Describe su solicitud → Envía ticket
→ El ticket se guarda en localStorage → El dashboard detecta el cambio automáticamente
→ La gestora recibe notificación (audio + browser) → Gestiona el ticket desde el dashboard

Estructura de Responsabilidades:
El sistema es Multi-Tenant (Multi-Área) simulado en frontend, actualmente con tres áreas activas:

1. Área de Gestión Empresarial (GE):
- Trámites Estructurales y Legales (Creación de empresas, RUT, representantes legales).
- Trámites Operativos (Actualización RUT, facturación electrónica, firmas).

2. Área de Gestión Humana (GH):
- Trámites de Personal y reclutamiento.
- Gestión de nómina y bienestar.

3. Área de Soporte TI (TI):
- Soporte técnico y fallas de sistemas.
- Administración global de cuentas de usuario, bloqueos y restablecimiento de claves.

🏗️ Estrategia de Desarrollo: "Local First"
Para garantizar una respuesta instantánea y autonomía en el diseño, utilizamos un sistema de Entorno de Pruebas Local.

1. Entorno Frontend (React / Vite)
- Archivos principales: SPA con rutas en `src/App.jsx` (`/login`, `/portal/:area`, `/dashboard/:area`, `/database/:area`).
- Arquitectura: Proyecto modularizado en React (Componentes Funcionales, Context API, Hooks).
- Persistencia (Desarrollo): Uso de un servicio de base de datos asíncrono (`DbService.js` y `SettingsManager.js`) que envuelven `localStorage` para guardar configuraciones, tickets, y usuarios.
- Comunicación entre pestañas: Evento "storage" del navegador para notificaciones en tiempo real, preparado para futura migración a WebSockets.

💻 Detalles Técnicos Clave
- Gestión Dinámica: Las solicitudes se gestionan mediante objetos de datos en localStorage.
- Indicadores Visuales: Uso de Canvas (sparklines) para mostrar tendencias de solicitudes.
- Vistas: Tabla de actividades con filtros + Vista Kanban (Pendiente / En progreso).
- Notificaciones: Audio con AudioContext (3 tonos) + Notificación nativa del navegador.
- Requisitos de Solicitud:
  - Descripciones específicas del trámite.
  - Para firmas: Incluir ruta en la unidad T.
  - Formato obligatorio: PDF.
