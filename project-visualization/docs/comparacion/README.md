# Documentación Técnica

Este directorio contiene la documentación técnica viva del proyecto.

## Ruta recomendada de lectura

1. `ARCHITECTURE.md`
   Explica la arquitectura general, los componentes principales y los diagramas conceptuales.

2. `TECHNICAL_SPECS.md`
   Detalla módulos, patrones, convenciones, flujos, riesgos y escalabilidad.

3. `DEPENDENCIES.md`
   Mapa de relaciones entre archivos y puntos de sincronización.

4. `DECISIONS.md`
   Decisiones de diseño que no deben “corregirse” sin contexto.

## Qué cubre esta documentación

- arquitectura actual del proyecto
- estructura modular
- flujo frontend -> persistencia -> actualización visual
- servicios principales
- manejo de estado
- riesgos técnicos
- buenas prácticas implementadas
- guía de entrada para futuras IAs

## Advertencia importante

Parte de la documentación histórica del repositorio describe estados anteriores. `ARCHITECTURE.md` y `TECHNICAL_SPECS.md` fueron redactados para reflejar el código real actual y deben tomarse como referencia principal.
